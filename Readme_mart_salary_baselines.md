CREATE TABLE mart.salary_baselines AS
SELECT 
    area_id, 
    experience_id,
    ROUND(AVG(
        CASE 
            WHEN salary_from IS NOT NULL AND salary_to IS NOT NULL THEN (salary_from::numeric + salary_to::numeric) / 2
            WHEN salary_from IS NOT NULL THEN salary_from::numeric
            WHEN salary_to IS NOT NULL THEN salary_to::numeric
        END
    )) as base_avg_salary,
    COUNT(*) as total_vacancies
FROM core.fact_vacancies
WHERE salary_currency = 'RUR'
GROUP BY area_id, experience_id;



Таблица mart.salary_baselines — это фундамент всей твоей аналитической модели. Если предыдущая таблица (с навыками) показывала «надстройку», то эта показывает «прожиточный минимум» рынка для конкретных условий.
Вот как правильно интерпретировать эту таблицу в описании исследования:
1. Смысловое назначение: Определение «точки нуля»
Эта таблица нивелирует влияние географии и стажа. Она отвечает на вопрос: «Сколько в среднем платят в этом городе человеку с таким-то опытом, независимо от его специфических навыков?»
Без этой таблицы мы бы сравнивали зарплату Junior-разработчика в Москве с зарплатой Senior-мерчендайзера в Барнауле, что привело бы к ложным выводам о «стоимости» их навыков.
2. Ключевые показатели (Метрики)
area_id + experience_id: Это твои контрольные группы. Разделение по ним позволяет очистить данные от «шума» региональных коэффициентов и карьерных ступеней.
base_avg_salary (Базовая стоимость): Это рыночная норма.
Если вакансия предлагает сумму выше этой базы — компания платит за редкие навыки или бренд.
Если ниже — это либо масс-маркет, либо вакансия с низкими требованиями.
total_vacancies (Вес сегмента): Индикатор достоверности.
Сегменты, где total_vacancies > 50, — это устойчивые рыночные стандарты.
Сегменты с total_vacancies < 5 — это зоны статистической погрешности (единичные уникальные вакансии), на которые нельзя опираться при долгосрочном планировании.
3. Пример интерпретации конкретных строк (из твоего лога):
Возьмем твои данные:
ID 88 / between3And6 / 152 271 руб. (12 вакансий): Это зрелый, стабильный сегмент. База в 152к — это «входной билет» для опытного специалиста в данной локации. Любой навык, который мы изучаем в этом сегменте, будет конкурировать с этой цифрой.
ID 19 / noExperience / 26 813 руб. (4 вакансии): Это сегмент старта карьеры. Здесь база очень низкая, поэтому даже небольшая доплата (например, +10 000 руб.) за владение Excel даст огромный profit_percent.
Резюме для отчета:
«Таблица базовых стоимостей salary_baselines служит эталоном (бенчмарком) для оценки эффективности навыков. Она позволяет перевести абсолютные значения зарплат в относительные, выявляя реальный вклад каждой компетенции в капитализацию специалиста. Расчет произведен на основе агрегированных данных о вилках зарплат (среднее значение from и to), что минимизирует субъективность предложений конкретных работодателей».
