CREATE TABLE mart.skill_costs AS -- Создаем таблицу из результата запроса
WITH cleansed_data AS (
    SELECT 
        vacancy_id, area_id, experience_id,
        CASE 
            WHEN salary_from IS NOT NULL AND salary_to IS NOT NULL THEN (salary_from::numeric + salary_to::numeric) / 2
            WHEN salary_from IS NOT NULL THEN salary_from::numeric
            WHEN salary_to IS NOT NULL THEN salary_to::numeric
            ELSE NULL 
        END AS avg_salary
    FROM core.fact_vacancies
    WHERE (salary_currency = 'RUR' OR salary_currency IS NULL)
      AND (salary_from IS NOT NULL OR salary_to IS NOT NULL)
),
market_avg AS (
    SELECT area_id, experience_id, AVG(avg_salary) as segment_avg_salary
    FROM cleansed_data
    GROUP BY area_id, experience_id
)
SELECT 
    LOWER(bridge.skill_name) as skill_name,
    COUNT(*) as count_vacancies,
    CAST(AVG(v.avg_salary) AS BIGINT) as avg_salary_total,
    CAST(AVG(v.avg_salary - m.segment_avg_salary) AS BIGINT) as skill_value_premium,
    CURRENT_DATE as calculation_date -- Чтобы знать, когда считали
FROM cleansed_data v
JOIN core.bridge_vacancy_skill bridge ON v.vacancy_id = bridge.vacancy_id
JOIN market_avg m ON v.area_id = m.area_id AND v.experience_id = m.experience_id
GROUP BY 1
HAVING COUNT(*) >= 3;



Таблица mart.skill_costs — это итоговая аналитическая витрина ценности компетенций. В отличие от базовых стоимостей (которые описывают рынок «в общем»), эта таблица описывает конкретные рычаги влияния на доход.
Вот как правильно интерпретировать её структуру и данные в описании:
1. Смысловое назначение: Индекс добавочной стоимости
Эта витрина отвечает на вопрос: «Сколько рынок готов доплачивать сверху за владение конкретным инструментом или знанием?»
Она переводит абстрактные требования вакансий в конкретные денежные эквиваленты.
2. Ключевые показатели (Метрики) и их трактовка
skill_name (Нормализованный навык): Единица анализа. Благодаря функции LOWER, мы объединили одинаковые навыки, написанные в разных регистрах (SQL, sql), что делает данные чистыми.
count_vacancies (Популярность/Спрос):
Показывает, насколько навык «стандартен».
Интерпретация: Если у навыка высокий count_vacancies и высокий skill_value_premium, это «золотой стандарт» индустрии. Если спрос низкий, а премия высокая — это «узкая ниша».
avg_salary_total (Средний чек вакансии):
Это «витринная» цена вакансий, где встречается навык.
Важно: Это число само по себе может вводить в заблуждение (например, в Москве оно всегда выше), поэтому мы используем следующую метрику.
skill_value_premium (Чистая стоимость навыка):
Самая важная метрика. Это разница между реальной зарплатой в вакансии и «базой» для этого города и опыта.
Интерпретация: Показывает «чистый вес» навыка. Если он равен +50 000, значит, этот навык — основной драйвер роста зарплаты в данной вакансии.
3. Логика расчета (Методология)
В описании стоит подчеркнуть, что расчет строится на сравнении внутри сегмента:
Мы берем вакансию с навыком (например, «Python» в Новосибирске, опыт 1–3 года).
Вычитаем из неё среднюю зарплату всех вакансий в Новосибирске с опытом 1–3 года.
Полученный остаток усредняем по всем упоминаниям навыка.
Это позволяет честно оценить навык, не смешивая «дорогие» города с «дешевыми» и «сеньоров» с «джунами».
4. Пример интерпретации результатов витрины:
Если в этой таблице мы видим:
sql: count=195, premium=16 263 — это массовый, стабильно востребованный навык, дающий умеренную, но гарантированную прибавку.
mlops: count=3, premium=139 729 — это дефицитный, высокоэкспертный навык. Его «чистая стоимость» колоссальна, но вакансий мало — рынок «штучный».
выкладка товаров: count=263, premium=-12 281 — навык-маркер низкооплачиваемых должностей. Его наличие в резюме (без других сильных навыков) удерживает доход ниже среднерыночного.
Резюме для документации:
«Витрина mart.skill_costs представляет собой рейтинг компетенций, ранжированных по их способности увеличивать рыночную стоимость кандидата. Метрика skill_value_premium позволяет выделить навыки, обладающие наибольшим инвестиционным потенциалом для обучения, так как она очищена от влияния региональных факторов и трудового стажа».
